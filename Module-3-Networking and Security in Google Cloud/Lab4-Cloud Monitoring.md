# Cloud Monitoring

## Introduction

This lab focuses on using Google Cloud Monitoring to observe the performance, availability, and health of cloud resources. Monitoring and logging agents were installed on a virtual machine to collect system metrics and logs. Additional monitoring features such as uptime checks, alerting policies, dashboards, and log analysis were configured to provide comprehensive visibility into resource performance.

## Objectives

* Create and configure a Compute Engine virtual machine.
* Install Monitoring and Logging agents.
* Monitor VM performance using Cloud Monitoring.
* Configure uptime checks to verify service availability.
* Create alerting policies for resource monitoring.
* Build custom dashboards and charts for metric visualization.
* Analyze logs using Cloud Logging.

## Key Steps Performed

1. Created a Compute Engine virtual machine instance named **lamp-1-vm**.
2. Installed and configured the Apache HTTP Server.
3. Verified web server accessibility through the instance's external IP address.
4. Enabled Cloud Monitoring and configured a Monitoring Metrics Scope.
5. Installed the Google Cloud Ops Agent for monitoring and logging.
6. Created an uptime check to monitor web server availability.
7. Configured an alerting policy based on network traffic metrics.
8. Added an email notification channel for alert delivery.
9. Created a custom dashboard with CPU Load and Received Packets charts.
10. Explored Cloud Logging and reviewed logs generated by VM operations.
11. Tested monitoring functionality by stopping and restarting the virtual machine.
12. Verified uptime check results and alert notifications.

## Results

The virtual machine was successfully monitored using Cloud Monitoring and Cloud Logging services. Monitoring and logging agents collected system metrics and log data effectively. Uptime checks provided continuous verification of service availability, while alerting policies generated notifications based on predefined thresholds. Custom dashboards displayed key performance metrics, and Cloud Logging captured operational events such as VM startup and shutdown activities.

## Conclusion

This lab provided practical experience with Google Cloud Monitoring and Cloud Logging tools. The implementation of monitoring agents, uptime checks, alerts, dashboards, and log analysis demonstrated how cloud resources can be observed and managed efficiently. These capabilities support proactive monitoring, faster issue detection, and improved operational visibility within cloud environments.
