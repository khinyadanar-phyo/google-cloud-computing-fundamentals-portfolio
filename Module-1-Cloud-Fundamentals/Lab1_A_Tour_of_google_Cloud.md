## A Tour of Google Cloud – Hands-on Lab
The aim of this lab is to understand the basics of Google Cloud Platform by exploring the Cloud Console, learning how projects are structured, managing access using IAM, and getting an introduction to Google Cloud APIs such as Dialogflow.

## Objective 
In this lab, I explored Google Cloud to understand how the platform works in practice. I learned how to navigate the Cloud Console, how different projects are organized, how user permissions are controlled using IAM, and how APIs like Dialogflow can be used to build simple conversational applications. The main purpose was to become familiar with the cloud environment and its basic services.

## Key Steps Performed
Logged into Google Cloud Console using lab credentials
Explored the main interface and navigation menu
Viewed different Google Cloud projects and understood their structure
Learned that projects are isolated environments for managing resources
Navigated through services like Compute Engine, Cloud Storage, IAM, and APIs
Checked IAM roles and permissions for different users
Opened the API Library and explored available APIs
Explored Dialogflow API and understood its basic purpose
Tools / Services Used
Google Cloud Console
Cloud IAM (Identity and Access Management)
Google Cloud Projects
API Library
Dialogflow API

## Result
By completing this lab, I gained a basic understanding of how Google Cloud works. I was able to navigate the console, understand project structure, explore IAM roles, and learn how APIs like Dialogflow are used to build conversational applications without needing to understand the underlying machine learning concepts.
